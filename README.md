# 🍽️ Restaurant Operations Analysis (SQL Project)

## 📌 Objective
Analyze restaurant performance using SQL queries to uncover insights about menu pricing, category distribution, customer order behavior, and spending patterns. The project demonstrates how SQL can drive business intelligence in the hospitality domain.

## 🛠️ Requirements
- SQL database: `restaurant_db`
- Tables: `menu_items`, `order_details`
- Skills: Joins, aggregations, filters, grouping, ordering
- Business questions:
  1. What items are least/most expensive?
  2. How many dishes exist in each category?
  3. What is the average price per category?
  4. What are the most/least ordered items?
  5. Which customers/orders generated the highest spend?

## 🔍 Solutions & Insights
1. **Menu Analysis**
   - Total items on menu identified.
   - Least & most expensive dishes revealed.
   - Italian dishes counted and priced.
   - Average price calculated per category.

2. **Order Behavior**
   - Date range of orders established.
   - Total orders and items counted.
   - Orders with >12 items highlighted.
   - Most item-heavy order identified.

3. **Customer Spend**
   - Top 5 highest-spend orders extracted.
   - Categories contributing most to revenue analyzed.
   - Least & most popular items revealed customer preferences.

## 📊 Tools Used
- **SQL (MySQL)**
- Database: `restaurant_db`
- Tables: `menu_items`, `order_details`

## ✅ Outcome
This project showcases the use of SQL for practical business intelligence in the restaurant industry. Insights gained can help optimize menu offerings, understand customer behavior, and improve operational strategy.

### 🚀 How to Run
1. Clone the repository.
2. Import the `restaurant_db` schema.
3. Run the SQL queries provided in `/queries`.
4. Review results for insights.
